## 一、原作者项目概况

基于原版源码分析：

| 项目 | 内容 |
|------|------|
| **项目名称** | cn.LammaForms |
| **开发框架** | .NET 10.0 (net10.0-windows) — Windows Forms 桌面应用 |
| **目标平台** | Windows (WinExe) |
| **支持的 llama.cpp 版本** | 基于 `b9596` 及之前版本（从注释和默认参数推断） |
| **编程语言** | C# 13 (隐式 using、nullable、顶级语句样式) |

## 二、我们做过哪些修改

### 2.1 核心新增功能

#### ① MTP (Multi-Token Prediction) 模型支持
- **AppConfig.cs**：新增 `MtpFile` 属性，存储 MTP 模型文件路径
- **mainFrm.Designer.cs**：新增 `label_mtpFile`、`tb_mtpFile`、`btn_mtpFile` 三个 UI 控件
- **mainFrm.cs**：
  - 加载/保存 MTP 文件路径
  - 切换模型时自动匹配 MTP 模型（自动查找同目录下包含 "MTP" 或 "assistant-MTP" 的 gguf 文件）
  - `AutoMatchMtpFile()` 方法实现智能匹配逻辑

#### ② 窗口分隔条位置持久化
- **AppConfig.cs**：新增 `SplitterDistance`（右侧分隔条位置）和 `SplitterDistanceLeft`（左侧分隔条位置）
- **mainFrm.Designer.cs**：整体布局从固定 Panel 改为 SplitContainer 布局
  - `splitContainer_main`：主垂直分隔（左/右面板）
  - `splitContainer_left`：左侧水平分隔（模型参数区/启动配置区）
  - `splitContainer`：右侧水平分隔（高级工具/日志区）
- **mainFrm.cs**：加载/保存/恢复分隔条位置

#### ③ 新增三大参数分类（v2.4.0+）
在 `SystemParamManager.cs` 的 `CreateDefaultParams()` 中新增：

| 序号 | 分类名称 | 图标 | 参数数 | 关键参数 |
|------|----------|------|--------|----------|
| **九** | 推理与聊天模板 | 🧠 | 11 个 | `--reasoning-format`, `--reasoning-budget`, `--chat-template`, `--system-prompt`, `--conversation` 等 |
| **十** | HuggingFace 一键加载 | 🤗 | 8 个 | `--hf-repo`, `--hf-file`, `--hf-token`, `--model-url`, `--docker-repo` 等 |
| **十一** | 特殊功能 | ✨ | 11 个 | `--context-shift`, `--cont-batching`, `--parallel`, `--rpc`, `--lora`, `--grammar`, `--json-schema` 等 |

#### ④ v2.4.0+ 新增参数扩充（补充到原有分类 1-8）
在已有 8 个分类中大量增加了新参数，尤其：

- **CPU 调度与亲和性**：`--cpu-moe`, `--swa-full`, `--warmup` 等
- **KV 缓存控制**：KV cache 量化相关参数
- **采样补充**：新增多种采样策略参数
- **设备调度**：`--op-offload`, `--no-host` 等
- **日志控制**：`--log-verbosity`, `--log-prefix`, `--log-timestamps`, `--offline` 等

#### ⑤ system.json 兼容性修复
- 新增 `FixKnownParamTypes()` 方法：加载旧版 system.json 时，将已知的文件路径参数（如 `--chat-template-file`, `--lora`, `--grammar-file` 等）从 `String` 类型修正为 `FilePath` 类型，确保显示浏览按钮

#### ⑥ 三态布尔参数扩展
- `SystemParamItem.cs`：`FlashAttn` 输出样式新增 `--fit` 和 `--reasoning` 参数
- FlagOnly 纯开关列表从约 16 个扩展到约 52 个（新增约 36 个 v2.4.0+ 纯开关参数）

### 2.2 UI 布局改动

| 改动项 | 原版 | 修改版 |
|--------|------|--------|
| **主布局** | 固定 Panel 布局 | SplitContainer 可拖拽分隔条布局 |
| **参数控件列数** | 4 列（CheckBox + Label + 值控件 + 提示标签） | **3 列**（去掉提示标签列，值控件占 70%） |
| **Label 宽度** | 28% | 30% |
| **值控件宽度** | 50% | 70% |
| **CheckBox 对齐** | `Anchor = Left` | `Anchor = Left \| Top`（顶部对齐更好） |
| **FilePath 输入框** | `ReadOnly = true`（只能通过浏览按钮选择） | `ReadOnly = false`（可手动编辑路径） |
| **FilePath 浏览按钮** | `Text = "..."`，宽度 36 | `Text = "浏览"`，宽度 50，新增 `FlatStyle.Standard` |
| **打开文件对话框** | 无指定父窗口 | 使用 `FindForm()` 指定父窗口（避免模态对话框错位） |

### 2.3 构建配置改动

| 改动项 | 原版 | 修改版 |
|--------|------|--------|
| **输出目录** | 默认 (bin/Debug/net10.0-windows/) | `..\\..\\`（输出到项目根目录上一级） |
| **目标框架后缀** | 追加到输出路径 | `AppendTargetFrameworkToOutputPath = false` |

### 2.4 删除的内容

- `SystemParamItem.cs`：删除了 `--no-affinity` 纯开关参数（llama.cpp b9596+ 已废弃）
- `ParamControlFactory.cs`：移除了每个参数行右侧的提示标签列（第4列 22%），减少水平冗余
- 旧版 `mainFrm.Designer.cs` 的固定 Panel 布局被 SplitContainer 方案完全取代

### 2.5 未修改的文件

| 文件 | 状态 |
|------|------|
| `Program.cs` | **完全一致** |
| `Config/ConfigManager.cs` | **完全一致** |

## 四、总结

**修改版相比原版的主要提升：**

1. **适配 llama.cpp b9596+ 新特性**：新增了推理/聊天模板、HuggingFace 一键加载、特殊功能三大参数分类，以及大量新参数
2. **MTP 多 Token 预测支持**：新增 MTP 模型文件选择和自动匹配功能
3. **UI 体验优化**：从固定 Panel 改为 SplitContainer 可拖拽布局，分隔条位置可持久化保存
4. **参数管理增强**：增加 system.json 兼容性修复，修正旧版本缓存中的参数类型
5. **控件交互改进**：FilePath 可编辑、浏览按钮中文标识、对话框父窗口修复
6. **代码行数净增约 1076 行**，功能大幅扩展

**与原版的兼容性：**
- system.json 旧缓存会被自动修正（`FixKnownParamTypes`）
- config.json 新增字段有默认值，加载旧配置不会报错
- Program.cs 和 ConfigManager.cs 完全未修改，入口逻辑保持不变
