# 2026-06-12 开发日志

## 今日修改汇总

### 1. MTP 模型自动加载（新增功能）
- **新增 `AppConfig.MtpFile` 字段** — 保存 MTP 模型文件路径到 config.json
- **新增 `AutoMatchMtpFile()`** — 选主模型时自动扫描同目录 `*MTP*` 文件，优先 `assistant-MTP`
- **3 处命令构造加入 `--model-draft`** — CMD对话测试 / 生成server脚本 / 开启web访问，自动拼接 MTP 参数
- 文件: `AppConfig.cs`, `mainFrm.cs`

### 2. 文件路径参数增加浏览按钮（7 个）
- `--chat-template-file`、`--system-prompt-file`
- `--log-file`
- `--lora`、`--control-vector`、`--grammar-file`、`--json-schema-file`
- 类型从 `String` → `FilePath`，自动显示可编辑文本框 + 浏览按钮
- 文件: `SystemParamManager.cs`

### 3. FilePath 控件修复（文本框可编辑 + 对话框绑定父窗口）
- `ReadOnly = true` → `false`，允许手动路径输入
- 增加 `TextChanged` 事件同步
- `FindForm()` 绑定对话框父窗口，解决弹窗不显示问题
- 按钮宽度 36→50px，文字 "..."→"浏览"
- 文件: `ParamControlFactory.cs`

### 4. FixKnownParamTypes 兼容旧缓存
- 新增 `FixKnownParamTypes()` 方法，在加载旧 `system.json` 后强制修正已知文件路径参数类型
- 文件: `SystemParamManager.cs`

### 5. 布局大改造：左右面板间加可拖动的分隔条
- 新增 `splitContainer_main`（水平方向，Dock=Fill）
- `panel_left` 从 `Dock=Left`（固定 684px）改为 `Dock=Fill`（在 splitter 内）
- `panel_right` 去掉 `Location` 硬编码，改为 `Dock=Fill`（在 splitter 内）
- 初始 `SplitterDistance = 684`，用户可自由拖动
- 文件: `mainFrm.Designer.cs`

### 6. 分隔线从 Paint 改为实体 Panel
- Paint 画的线被 Dock=Fill 的 tlp 控件盖住不可见
- 改为 1px 高的实体 Panel（Dock=Bottom），后添加确保高 Z-Order 先锚定
- 文件: `ParamControlFactory.cs`

### 7. 布局列宽调整 + 去掉冗余提示栏
- 列宽从 4 列（25px/28%/50%/22%）改为 **3 列**（25px/30%/70%）
- 去掉右侧灰色描述文字（鼠标悬停 ToolTip 已有完整说明，不必重复显示占空间）
- 文件: `ParamControlFactory.cs`

### 8. 分类标题箭头对齐修复
- 箭头字体从 `Microsoft YaHei UI 9F` → `Segoe UI Emoji 10F`，与左图标统一
- 文件: `ParamControlFactory.cs`

### 9. 分隔条默认值改为用户实际值
- 右侧分离条：470→**427**（从 config.json 读取）
- 左侧分离条：537→**433**（从 config.json 读取）
- 文件: `AppConfig.cs`
