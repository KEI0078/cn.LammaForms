## 今日修改汇总

### 1. v2.4.1 三个隐藏 bug 修复（commit `6b90a38`）

#### B-1：批量测速缺漏 `--model-draft`
- **问题**：批量测速只从 `_paramManager` 动态参数面板捞参数，完全绕过独立 `tb_mtpFile` 控件
- **症状**：启用了 `--spec-type draft-mtp` 类参数，但命令 echo 不含 `--model-draft "..."mtp.gguf"`，MTP 静默失效
- **修复**（`mainFrm.cs:2403`）：在批量测速拼 `fullArgs` 时补上 MTP draft model 路径，跟 chatTest / saveBatchCmd / openclaw_web 三个位置保持一致
- **诊断特征**：单条 chatTest echo 有 `--model-draft`，批量测速 echo 没有
- **根因类别**："UI 控件存在 ≠ 逻辑已对接"（架构原则 #7）

#### B-2：b9601+ llama-cli 速度输出格式不匹配
- **问题**：`ParseSpeedLine` / `ParseSpeedFromOutput` 匹配老格式 `Prompt: 1234.5 t/s | Generation: 56.6 t/s`
- **症状**：批量测速 4 档全显示 0.0 t/s，右上角 token 计数器全 0
- **修复**（`mainFrm.cs:2586` + `mainFrm.cs:2642`）：重写两个解析方法，匹配 b9601 结构化输出 `prompt eval time = X ms / Y tokens`，把 `tokens / (ms/1000)` 换算成 t/s
- **坑中坑**：`Split(new[] { '\n', '\r' }, ...)` 这种字面 `'\r'` 在 patch 工具下被破坏，导致编译 8 个 `CS1010/1011/1003` 错。**改用 `new char[] { '\n', (char)13 }`** 绕开（详见 SKILL.md 陷阱 #1 升级版）

#### B-3：CMD 窗口对话测试起不来（Win32Exception "操作已被用户取消"）
- **问题**：`btn_chatTest_Click` 用 `UseShellExecute = true` 起 CMD 窗口，工作目录 `E:\llama.cpp\启动管理工具cn.Lamma-windows v2.4`（含 CJK + 长路径）
- **症状**：Win11 24H2 下 ShellExecuteEx 返回 ERROR_CANCELLED (1223)
- **修复**（`mainFrm.cs:1635`）：`UseShellExecute = false` + `CreateNoWindow = false` + `WorkingDirectory = Path.GetDirectoryName(cliExe)`（绕开中文路径当工作目录，改到 `E:\llama.cpp\llama-b9601-bin-win-cuda-13.3-x64`）

### 6. MTP 闪退配置避坑（非代码 bug，已写进 CHANGELOG）

- **触发**：v2.4.1 部署后主人实测，CMD 闪退
- **主人实测工作流**（成功路径）：
  1. 切到 `gemma-4-12B-it-qat-UD-Q4_K_XL.gguf`（带 MTP head 的版本）
  2. 取消"八、调试与日志参数"分类的所有勾选（去掉 `--perplexity` 等冲突项）
  3. KV 缓存类型用 f16，ctx-size 4096
  4. 速度 `Prompt: 44.5 t/s | Generation: 76.7 t/s`（MTP 加速生效）
- **闪退根因总结**：
  - **真因 1**：Q4_0（无 MTP head）+ mtpFile 填了 + 启 `--spec-type draft-mtp` → `Gemma4Assistant requires ctx_other to be set` 错误
  - **真因 2**：`--perplexity` 模式跟 chatTest 互斥——llama-cli 立即计算退出
  - **真因 3**：`--log-file` 路径含中文/空格 + 启调试参数 → 偶尔 Win32Exception
- **v2.4 看似"正常"的真相**：v2.4 用的是 `--single-turn -1` 单行预设（不启用 MTP），拼出命令立刻 exit 0，**不是真"打开"**
- **v2.4.1 vs v2.4 的 .exe/.dll md5 完全一致**——**不是 v2.4.1 代码 bug**

## 关键经验教训（小 A 自省）

1. **"UI 控件存在 ≠ 逻辑已对接"**——B-1 根因。4 个"陪伴文件"命令构造位置里有 3 个正确处理 `tb_mtpFile`，**唯独批量测速漏了**。**给陪伴文件类型加新命令构造位置时必须 grep 全部位置**，不能只信表格。
2. **`'\r'` 字面 char 在 patch 工具下被破坏**——B-2 修法踩的坑。**用 `(char)13` 替代字面 `'\r'`** 完全避开。SKILL.md 陷阱 #1 升级版已记。
3. **配置 + 路径组合可能让同一份代码表现出不同症状**——B-4 教训。主人说"勾 KV 缓存到 GPU 才闪退"实际是错觉（不勾也闪退，只是退出太快看不清）。**调试时不能只听症状，要看实际拼出的命令**。
4. **"看着像垃圾但可能是历史备份"的文件**——v2.4.0 隐藏 19 个 commit 的 `.git_old/` 差点被当垃圾删。**先列清单 + 分类 + 问主人拍板**，不要按"自动清理"思路直接 rm。
5. **主人说的"v2.4.0 时代能点"可能是错觉**——B-5 教训。Designer.cs 自 v2.4.0 以来一行没改过，4 个独立按钮在 v2.4.0 也点不开，只是主人之前没用过。**主人描述"v2.4.0 能用"不一定是真，要核对代码**。
